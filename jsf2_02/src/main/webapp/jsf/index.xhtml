JSF 2.2 Whirlwind Tour

This section gives a quick example of each of the most basic JSF 2.2 capabilities. This tutorial section gives only brief examples with very limited explanation, but each of the topics is covered in detail in later sections of the JSF 2.2 tutorial and the PrimeFaces tutorial. Also, the topics given here are only the most basic ones: the full tutorial covers many other important but not-quite-as-fundamental topics such as looping with ui:repeat, data tables, composite components (four sections!), property files, I18N, GUI events, view parameters, and so on.

Click on one of the links below to get a quick example of the capability.

  • setup.jsf -- Set up your IDE and project to use JSF 2.2.
    For details, see the "Installation, Setup, and Getting Started" section.
  • test-tags.jsf -- Verify that h:commandButton and h:inputText are rendered as a button and a textfield, respectively.
    For details, see the "Installation, Setup, and Getting Started" section.
  • navigate.jsf -- Use Java to navigate to a results page. Uses the return value of the "action" method as the base name of the page to navigate to.
    For details on declaring beans and implicit navigation (where the return value of the action controller method is the base name of the results page), see the "Programming Basics" section. For fancier navigation that uses rules defined in faces-config.xml, see the section entitled "Explicit Page Navigation and faces-config.xml".
  • bean-properties.jsf -- Represent form data with bean properties.
    For details on beans, see the Managed Beans 1 and Managed Beans 2 sections.
  • business-logic.jsf -- Apply business logic to calculate results that are derived from values passed in by user.
    For details, see the banking example of the Managed Beans 1 section.
  • page-templating.jsf -- Use page templates and include files to avoid repeating facelets code.
    For details, see the Page Templating section.
  • validate-data.jsf -- Validate the input data, and display error messages on input form if data is in wrong format.
    For details, see the section on validating user input.
  • ajax.jsf -- Ajaxify the pages so that results can be inserted into current page without doing a complete page reload.
    For details, see the Ajax section.
  • primefaces.jsf -- Make your GUI elements prettier by using PrimeFaces.
    For details, see the separate PrimeFaces tutorial series.
